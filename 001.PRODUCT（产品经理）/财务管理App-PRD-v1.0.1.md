## 1. 产品概述

### 1.1 产品背景

记录每日消费情况，了解钱花在哪里，培养良好的消费习惯，实现财务健康。

### 1.2 产品定位

简洁、轻量的个人记账工具，专注于日常消费记录与统计。

### 1.3 目标用户

- 有记账习惯的个人
- 希望控制消费的用户
- 追求简单操作的用户

---

## 2. 用户需求

### 2.1 核心需求

| 需求 | 描述 | 优先级 |
|------|------|--------|
| 用户注册登录 | 用户名+密码方式认证 | P0 |
| 快速记账 | 3秒内完成一笔记录 | P0 |
| 查看统计 | 了解消费分布 | P0 |
| 消费趋势 | 按日/周/月查看变化 | P1 |
| 数据安全 | 记录不丢失，支持数据同步 | P1 |

### 2.2 用户痛点

- 现有记账软件功能复杂，使用成本高
- 记完账不知道花在哪
- 换设备数据丢失
- 多设备使用需求

## 3. 功能列表

### 3.1 用户认证

#### 3.1.1 注册功能
- **用户名**：唯一标识，4-20位字母或数字
- **密码**：6位以上，安全可靠
- **确认密码**：二次确认，防止误输入
- **注册成功**：自动登录并跳转首页

#### 3.1.2 登录功能
- **用户名+密码**：简洁的登录方式
- **记住登录状态**：7天内免登录
- **登录成功**：跳转首页并加载用户数据

#### 3.1.3 登出功能
- 顶部菜单提供登出选项
- 登出后清除本地登录状态
- 返回登录页面

### 3.2 核心功能

#### 3.2.1 记账功能
- **快速记账入口**：首页一键添加
- **选择分类**：餐饮、交通、购物、娱乐、居住、医疗、教育、其他
- **输入金额**：数字键盘
- **备注**：可选，简短描述

#### 3.2.2 记录列表
- 按日期分组展示消费记录
- 支持删除记录
- 下拉刷新

#### 3.2.3 统计报表
- **饼图**：各类别消费占比
- **柱状图**：近7天消费趋势
- **月度汇总**：本月总消费、平均日消费

#### 3.2.4 分类管理
- 系统预设8个分类
- 每个分类配有图标和颜色

### 3.3 交互规则

| 场景 | 交互 |
|------|------|
| 未登录状态 | 进入登录/注册页面 |
| 已登录状态 | 进入首页 |
| 首页点击"记一笔" | 弹出记账表单 |
| 选择分类 | 点击图标选中，支持切换 |
| 确认金额 | 点击完成按钮 |
| 删除记录 | 左滑显示删除按钮 |
| 查看更多统计 | 横向滑动统计区域 |
| 点击头像/用户名 | 显示下拉菜单（修改密码、登出） |

## 6. 数据需求

### 6.1 数据字段

#### 用户表 (users)
| 字段 | 类型 | 说明 |
|------|------|------|
| id | string | 唯一标识 |
| username | string | 用户名（唯一） |
| password | string | 密码（加密存储） |
| createdAt | timestamp | 创建时间 |

#### 消费记录表 (records)
| 字段 | 类型 | 说明 |
|------|------|------|
| id | string | 唯一标识 |
| userId | string | 用户ID |
| amount | number | 消费金额 |
| category | string | 分类ID |
| remark | string | 备注 |
| createdAt | timestamp | 创建时间 |

### 6.2 本地存储

- **LocalStorage 存储结构**：
  - `currentUser`: 当前登录用户信息
  - `finance_records_{userId}`: 按用户ID存储的记录
- **登录状态**：7天有效期的 token 或记住登录状态

### 6.3 密码安全

- 密码使用 SHA-256 或更安全的加密方式存储
- 传输过程使用 HTTPS（上线后）

## 8. 非功能性需求

| 需求 | 说明 |
|------|------|
| 性能 | 首页加载 < 1秒 |
| 兼容性 | 现代浏览器（Chrome、Safari、Firefox） |
