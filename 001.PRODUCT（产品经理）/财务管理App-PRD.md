## 1. 产品概述

### 1.1 产品背景

记录每日消费情况，了解钱花在哪里，培养良好的消费习惯，实现财务健康。

### 1.2 产品定位

简洁、轻量的个人记账工具，专注于日常消费记录与统计。

### 1.3 目标用户

- 有记账习惯的个人
- 希望控制消费的用户
- 追求简单操作的用户

---

## 2. 用户需求

### 2.1 核心需求

| 需求 | 描述 | 优先级 |
|------|------|--------|
| 快速记账 | 3秒内完成一笔记录 | P0 |
| 查看统计 | 了解消费分布 | P0 |
| 消费趋势 | 按日/周/月查看变化 | P1 |
| 数据安全 | 记录不丢失 | P1 |

### 2.2 用户痛点

- 现有记账软件功能复杂，使用成本高
- 记完账不知道花在哪
- 换手机数据丢失

---

## 3. 功能列表

### 3.1 核心功能

#### 3.1.1 记账功能
- **快速记账入口**：首页一键添加
- **选择分类**：餐饮、交通、购物、娱乐、居住、医疗、教育、其他
- **输入金额**：数字键盘
- **备注**：可选，简短描述

#### 3.1.2 记录列表
- 按日期分组展示消费记录
- 支持删除记录
- 下拉刷新

#### 3.1.3 统计报表
- **饼图**：各类别消费占比
- **柱状图**：近7天消费趋势
- **月度汇总**：本月总消费、平均日消费

#### 3.1.4 分类管理
- 系统预设8个分类
- 每个分类配有图标和颜色

### 3.2 交互规则

| 场景 | 交互 |
|------|------|
| 首页点击"记一笔" | 弹出记账表单 |
| 选择分类 | 点击图标选中，支持切换 |
| 确认金额 | 点击完成按钮 |
| 删除记录 | 左滑显示删除按钮 |
| 查看更多统计 | 横向滑动统计区域 |

## 4. 信息结构

```
App
├── 首页
│   ├── 今日消费概览
│   ├── 快速记账按钮
│   └── 最近消费记录（最近5条）
├── 记录页
│   └── 全部消费记录（按日分组）
└── 统计页
    ├── 本月消费概览
    ├── 消费分类饼图
    └── 近7天趋势柱状图
```

## 6. 数据需求

### 6.1 数据字段

| 字段 | 类型 | 说明 |
|------|------|------|
| id | string | 唯一标识 |
| amount | number | 消费金额 |
| category | string | 分类ID |
| remark | string | 备注 |
| createdAt | timestamp | 创建时间 |

### 6.2 本地存储

- 使用浏览器 LocalStorage 存储
- 数据结构：`{ records: [...], lastSync: timestamp }`
